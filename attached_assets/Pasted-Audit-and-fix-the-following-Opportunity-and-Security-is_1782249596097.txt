ISSUE #2 – Participant Count Override Save Error

Current error:

This usually indicates the frontend expects JSON but the backend is returning HTML due to a failed route, redirect, exception, or invalid API response.

Audit:

- Opportunity save endpoint
- Participant Count Override field
- Form validation
- API response format
- Database insert/update queries

Requirements:

- Participant Count Override should save correctly.
- Accept integer values only.
- Blank value should auto-calculate participants.
- API must always return JSON.
- Never return HTML to JSON requests.

Add proper error handling:

{
"success": false,
"message": "Descriptive error"
}

instead of server HTML pages.

ISSUE #3 – 2FA Withdrawal Logic

Current behavior:
2FA is being treated as mandatory.

Required behavior:

2FA should be OPTIONAL.

Withdrawal Security Flow:

1. User enters withdrawal password.

IF withdrawal password is correct:
→ Process withdrawal normally.

IF withdrawal password is incorrect:
→ Ask for 2FA code as secondary verification.

OR

Admin setting:

Enable:
"Require 2FA For Withdrawals"

Options:

- Always Require
- Optional (Default)
- Disabled

Default setting:
Optional
